# Storybook — Des liens accessibles

Source : `source-liens-accessibles-v4.md`

Objectif : transformer les règles de liens accessibles en une série courte de slides Imagegen, visuelles, sobres et utilisables en formation accessibilité.

## Masque commun v3

- Format image 16:9, fond blanc, panneaux gris froid, bleu institutionnel dominant.
- Aucun repère de progression visible, aucun compteur `1/6` ou équivalent.
- Titre large en haut gauche, sous-titre court, scène centrale et callout bas.
- Personnages plats sobres et objets concrets : panneau d’orientation, liste de liens isolée, table de correction, lecteur d’écran, clavier, carte image-lien.
- Aucun logo, aucune photographie, aucun dégradé, aucune décoration gratuite.

## Colonne vertébrale

1. Annoncer clairement la destination.
2. Tester le lien hors contexte.
3. Remplacer les intitulés vagues.
4. Prévenir les comportements particuliers.
5. Ne pas dépendre seulement de la couleur.
6. Nommer l’action quand une image porte le lien.

## Storyboard des slides

### Slide 1 — Un lien doit annoncer sa destination

Scène : une personne consulte un panneau d’orientation numérique où les liens vagues deviennent des destinations explicites.

Transformation : `Cliquez ici`, `En savoir plus` et `Lire la suite` laissent place à `Livre blanc accessibilité`, `Conditions de participation` et `Article liens accessibles`.

Message : le lien doit dire où il mène ou quelle action il déclenche.

Callout : `Un lien clair indique la destination avant le clic.`

### Slide 2 — Le lien doit rester clair hors contexte

Scène : deux fenêtres de lecteur d’écran comparent une liste isolée floue et une liste isolée claire.

Transformation : les intitulés répétés disparaissent au profit d’une liste autonome et compréhensible.

Message : la destination doit rester visible quand les liens sont lus seuls.

Callout : `Un lien lu seul doit encore dire où il mène.`

### Slide 3 — Remplacer les intitulés vagues

Scène : une table de correction compare une colonne `À éviter` et une colonne `À privilégier`.

Transformation : chaque lien vague reçoit une version plus précise dans une carte validée.

Message : la précision utile doit entrer dans le texte cliquable.

Callout : `La correction déplace l’information dans le lien.`

### Slide 4 — Prévenir les comportements particuliers

Scène : une agente contrôle deux portes d’action : téléchargement et nouvelle fenêtre.

Transformation : un lien muet reçoit `PDF, 2 Mo`, l’autre reçoit `nouvelle fenêtre`.

Message : l’utilisateur doit savoir ce qui va se passer avant d’activer le lien.

Callout : `Le comportement particulier s’annonce dans l’intitulé.`

### Slide 5 — Le lien ne doit pas dépendre uniquement de la couleur

Scène : une page de texte dense est testée avec et sans couleur, puis au clavier.

Transformation : un lien seulement bleu reçoit un soulignement et un état de focus visible.

Message : un lien doit rester identifiable sans dépendre d’un seul indice visuel.

Callout : `Couleur, soulignement et focus se complètent.`

### Slide 6 — Une image-lien décrit l’action

Scène : une icône maison sert de lien dans une maquette, avec le libellé accessible visible à côté.

Transformation : la description `maison` est remplacée par `Retour à l’accueil`.

Message : quand l’image porte le lien, l’alternative devient l’intitulé du lien.

Callout : `Décrire l’action utile, pas seulement le dessin.`
