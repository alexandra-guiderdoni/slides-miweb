# Storyboard v2 — Émojis accessibles

Source : `source-emojis-accessibles.md`

## Intention

Produire 8 slides image-based pour une formation à l'accessibilité numérique destinée à des communiquants. La série explique comment utiliser les émojis et les caractères Unicode dans les publications de réseaux sociaux sans rendre le message moins accessible.

## Masque commun

- Format 16:9 paysage.
- Aucun repère de progression visible : pas de compteur, pastille d'étape ou numéro de slide.
- Fond blanc ou gris très clair, avec beaucoup d'air.
- Titre large en haut gauche, sous-titre court juste dessous.
- Scène principale au centre : comparaison, flux ou transformation visible.
- Callout court en bas, sans footer.
- Palette sobre : bleu institutionnel `#001070`, gris froid, rouge `#E44850` seulement pour une alerte ou une erreur.
- Style : illustration plate institutionnelle, personnages sobres, icônes filaires, aucun logo de réseau social, aucun emblème officiel, aucune photographie, aucun dégradé, aucune 3D.
- Fidélité au texte : conserver les titres, sous-titres, labels et callouts fournis ; ne pas ajouter de texte parasite.

## Colonne vertébrale

1. Publier sans exclure : le message doit atteindre plusieurs publics.
2. Comprendre que les émojis sont vocalisés : le visuel devient du texte entendu.
3. Limiter la répétition : un seul signal suffit.
4. Garder les mots et chiffres en texte : l'information essentielle reste fiable.
5. Placer l'émoji après l'information : l'ordre de lecture réduit l'effort.
6. Bannir le faux gras : une apparence de mise en forme peut casser le texte.
7. Mettre en valeur autrement : la rédaction remplace les artifices Unicode.
8. Relire avant de publier : l'accessibilité devient un geste de production.

## Slides

### Slide 1 — Communiquer sans exclure

Rôle dans la colonne vertébrale : poser l'enjeu humain.
Idée principale : une publication sociale doit rester compréhensible quand l'effet visuel disparaît.
Scène : une table éditoriale vue de face ; à gauche, une personne prépare une publication visuelle ; au centre, le même contenu est simplifié en message clair ; à droite, une personne écoute le message via un lecteur d'écran et un canal accessible le reprend.
Personnage ou objet-système : poste de community manager, lecteur d'écran stylisé, canal accessible représenté par une page web générique et un affichage sans logo.
Transformation visible : publication décorative -> message clair -> accès par canal accessible.
Texte exact : titre `Communiquer sans exclure`, sous-titre `Le message reste prioritaire sur l'effet visuel.`, labels `Publication`, `Message clair`, `Lecteur d'écran`, `Canal accessible`, callout `L'apparence ne doit jamais porter seule l'information.`
Ce que l'image seule doit faire comprendre : le même message atteint plusieurs publics quand il reste textuel et clair.
Contraintes critiques : aucun logo, aucun numéro de progression, conserver le texte exact, ne pas suggérer que les émojis sont interdits.
Risque à éviter : scène trop technique, texte dense, faux logo de plateforme.

### Slide 2 — Un émoji est aussi vocalisé

Rôle dans la colonne vertébrale : expliquer le mécanisme invisible.
Idée principale : un émoji décoratif peut devenir du texte entendu.
Scène : une publication avec petits pictogrammes décoratifs entre dans un lecteur d'écran stylisé ; à la sortie, une bulle audio allongée montre que la synthèse ajoute des mots qui ralentissent l'écoute.
Personnage ou objet-système : publication simplifiée, lecteur d'écran, bulle audio.
Transformation visible : signe visuel -> équivalent textuel -> vocalisation plus longue.
Texte exact : titre `Un émoji est aussi vocalisé`, sous-titre `La synthèse vocale lit son équivalent textuel.`, labels `Émoji`, `Texte utile`, `Vocalisation`, `Mots parasites`, callout `Ce qui amuse à l'écran peut encombrer l'écoute.`
Ce que l'image seule doit faire comprendre : le lecteur d'écran transforme le symbole en mots.
Contraintes critiques : ne pas écrire de long verbatim, ne pas afficher de code, aucun numéro de progression, conserver le texte exact.
Risque à éviter : texte minuscule, confusion avec une interface réelle de plateforme.

### Slide 3 — Un seul émoji suffit

Rôle dans la colonne vertébrale : donner la première règle concrète.
Idée principale : répéter un émoji n'ajoute pas d'information.
Scène : deux publications sont comparées sur une même table ; à gauche, le message est entouré de signes d'alerte rouges et l'écoute devient bruyante ; à droite, un seul signal accompagne le message et la lecture reste nette.
Personnage ou objet-système : deux cartes de publication, un repère d'écoute simple.
Transformation visible : bruit visuel -> signal unique -> message lisible.
Texte exact : titre `Un seul émoji suffit`, sous-titre `La répétition allonge la vocalisation.`, labels `À éviter`, `Répétition`, `À privilégier`, `Signal unique`, callout `L'attention vient du message, pas de l'empilement.`
Ce que l'image seule doit faire comprendre : la version simple garde le sens sans empiler les signes.
Contraintes critiques : rouge uniquement sur l'erreur ou l'alerte, aucun numéro de progression, conserver le texte exact.
Risque à éviter : rouge décoratif excessif, vrai bouton d'urgence, accumulation qui rend la slide illisible.

### Slide 4 — Garder les mots et chiffres en texte

Rôle dans la colonne vertébrale : éviter les contresens.
Idée principale : les mots et les nombres essentiels doivent rester en texte réel.
Scène : une carte statistique cassée affiche deux touches décoratives `4` ambiguës ; une flèche de correction mène à une carte lisible où `44 %` est écrit comme du texte réel.
Personnage ou objet-système : carte statistique, lecteur d'écran stylisé, panneau de correction.
Transformation visible : symbole ambigu -> contresens -> texte fiable.
Texte exact : titre `Garder les mots et chiffres en texte`, sous-titre `Un émoji ne remplace pas une information.`, labels `Ambigu`, `44 %`, `Texte réel`, `Compréhensible`, callout `Les chiffres et les mots essentiels doivent rester écrits.`
Ce que l'image seule doit faire comprendre : remplacer un chiffre par un pictogramme casse la donnée.
Contraintes critiques : `44 %` doit apparaître exactement, aucun autre chiffre, aucun numéro de progression, conserver le texte exact.
Risque à éviter : statistique inventée, texte de carte trop long, confusion entre exemple et vraie donnée.

### Slide 5 — Placer l'émoji après le message

Rôle dans la colonne vertébrale : organiser la lecture.
Idée principale : l'information doit être entendue avant l'ornement.
Scène : deux lignes de publication deviennent une chronologie d'écoute ; en haut, des signes décoratifs bloquent le départ ; en bas, la phrase commence par le message et l'ornement arrive en fin de ligne.
Personnage ou objet-système : ligne de publication, chronologie de lecture, lecteur d'écran stylisé.
Transformation visible : obstacle avant le texte -> lecture directe -> ornement final.
Texte exact : titre `Placer l'émoji après le message`, sous-titre `L'information passe d'abord.`, labels `Début`, `Milieu`, `Fin`, `Lecture fluide`, callout `Laissez le lecteur d'écran annoncer le message avant l'ornement.`
Ce que l'image seule doit faire comprendre : l'ordre de lecture change l'effort.
Contraintes critiques : peu de signes visuels, aucun numéro de progression, conserver le texte exact.
Risque à éviter : phrase trop longue, trop d'émojis, rendu trop ludique.

### Slide 6 — Le faux gras n'est pas du gras

Rôle dans la colonne vertébrale : élargir le sujet des émojis aux caractères Unicode détournés.
Idée principale : les générateurs de faux gras ne mettent pas le texte en forme, ils remplacent les lettres par d'autres caractères.
Scène : à gauche, une publication utilise un mot en faux gras ; au centre, un générateur externe transforme les lettres ; à droite, trois sorties montrent les conséquences pour un lecteur d'écran, la recherche et la traduction.
Personnage ou objet-système : carte de publication, bloc de transformation Unicode, lecteur d'écran stylisé, loupe de recherche et bulle de traduction.
Transformation visible : texte réel -> caractères détournés -> perte de fiabilité.
Texte exact : titre `Le faux gras n'est pas du gras`, sous-titre `Des lettres remplacées peuvent casser le texte.`, labels `Faux gras`, `Caractères Unicode`, `Lecteur d'écran`, `Recherche`, `Traduction`, callout `Un effet visuel ne doit pas remplacer une vraie mise en forme.`
Ce que l'image seule doit faire comprendre : l'apparence de gras cache une transformation technique qui peut rendre le contenu moins accessible.
Contraintes critiques : ne pas afficher de vrai logo, ne pas utiliser de longue phrase en faux gras, garder les exemples courts, aucun numéro de progression, conserver le texte exact.
Risque à éviter : donner l'impression que tout Unicode est interdit, afficher du texte trop petit ou multiplier les effets décoratifs.

### Slide 7 — Mettre en valeur sans casser le texte

Rôle dans la colonne vertébrale : donner les alternatives opérationnelles au faux gras et aux décorations Unicode.
Idée principale : la mise en valeur accessible passe par l'ordre, la clarté et le texte réel.
Scène : deux colonnes comparent une publication décorée à proscrire et une publication claire ; la version à privilégier place l'information forte au début et garde une structure simple.
Personnage ou objet-système : deux cartes éditoriales, panneau de correction, liste de solutions.
Transformation visible : décoration fragile -> rédaction claire -> message lisible.
Texte exact : titre `Mettre en valeur sans casser le texte`, sous-titre `L'information forte reste en mots réels.`, labels `À éviter`, `À privilégier`, `Message d'abord`, `Texte réel`, `Pas de fausses colonnes`, `Pas de décor Unicode`, callout `La clarté remplace les artifices.`
Ce que l'image seule doit faire comprendre : on peut attirer l'attention sans transformer les caractères.
Contraintes critiques : éviter les blocs entiers en majuscules, ne pas créer de fausse colonne illisible, aucun numéro de progression, conserver le texte exact.
Risque à éviter : faire croire que les guillemets sont obligatoires, transformer la slide en cours de typographie.

### Slide 8 — Relire avant de publier

Rôle dans la colonne vertébrale : conclure par un geste de production.
Idée principale : l'accessibilité se vérifie avant diffusion.
Scène : un poste de contrôle éditorial ; une checklist courte valide la publication, une charte est posée à côté, puis trois canaux génériques partent vers une publication sociale sans logo, un site accessible et un affichage.
Personnage ou objet-système : bureau de contrôle éditorial, checklist, charte, canaux de diffusion génériques.
Transformation visible : publication prête -> contrôle -> diffusion accessible.
Texte exact : titre `Relire avant de publier`, sous-titre `Une charte simple sécurise les publications.`, labels `Sens`, `Nombre`, `Position`, `Texte réel`, `Pas de faux gras`, `Canal accessible`, callout `La publication accessible se relit avant diffusion.`
Ce que l'image seule doit faire comprendre : publier accessible demande une vérification courte mais systématique.
Contraintes critiques : checklist limitée aux six labels, aucun logo, aucune promesse de conformité automatique, aucun numéro de progression, conserver le texte exact.
Risque à éviter : checklist trop longue, faux logo, scène administrative froide sans action visible.
