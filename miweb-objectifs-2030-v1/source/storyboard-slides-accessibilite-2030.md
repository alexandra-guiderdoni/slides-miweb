## Colonne vertébrale

1. Montrer l’ambition 2030.
2. Rendre visible la dette qui revient quand l’accessibilité arrive trop tard.
3. Installer le bon périmètre : contribution SNUM, leviers MEF à consolider.
4. Déplier les trois axes opérationnels.
5. Faire apparaître les clarifications, les moyens et les indicateurs sans promettre trop tôt.

## Slide 1 - Installer l’objectif 2030

- Section illustrée : `Objectif 2030`.
- Rôle dans la colonne vertébrale : ouvrir sur l’ambition.
- Idée principale : l’accessibilité devient un marqueur de qualité numérique durable.
- Scène concrète visible : un guichet numérique public au centre, avec plusieurs usagers et agents qui accèdent au même service par des chemins différents.
- Personnage ou objet-système : guichet numérique partagé.
- Point de vue incarné : usager et agent.
- Transformation visible : service isolé -> service commun accessible -> trajectoire 2030.
- Texte exact : titre `Faire de l’accessibilité une qualité durable` ; labels `Usagers`, `Agents`, `Services numériques`, `Horizon 2030` ; callout `Une trajectoire de qualité, pas une promesse générale`.
- Lexique de slide : titre-action ; labels précis. Mots évités : `qualité` seule -> `qualité durable`, `ambition` seule -> `horizon 2030`.
- Ce que l’image seule doit faire comprendre : l’accessibilité concerne l’accès réel au service public.
- Alternative textuelle provisoire : visuel d’un guichet numérique public commun, utilisé par des usagers et des agents, placé sur une trajectoire vers 2030.
- Lien : ouvre le sujet ; prépare la dette qui se reconstitue quand l’accessibilité arrive trop tard.
- Risque à éviter : scène trop marketing ou promesse de conformité totale.

## Slide 2 - Montrer la dette qui revient

- Section illustrée : `Objectif 2030`.
- Rôle dans la colonne vertébrale : matérialiser le problème.
- Idée principale : corriger trop tard recrée de la dette à chaque évolution.
- Scène concrète visible : une chaîne de livraison où un audit arrive après la publication ; des étiquettes de non-conformités reviennent après une refonte.
- Personnage ou objet-système : chaîne de livraison numérique.
- Point de vue incarné : équipe projet.
- Transformation visible : audit tardif -> correction fragile -> dette qui revient.
- Texte exact : titre `Arrêter de corriger trop tard` ; labels `Audit final`, `Refonte`, `Régression`, `Correction priorisée` ; callout `Le risque : une dette qui se reconstitue`.
- Lexique de slide : titre-action ; labels concrets. Mots évités : `pilotage` -> `correction priorisée`, `amélioration` -> `régression détectée`.
- Ce que l’image seule doit faire comprendre : une correction placée en bout de chaîne ne tient pas dans le temps.
- Alternative textuelle provisoire : chaîne de livraison montrant un audit en fin de parcours, puis des régressions qui réapparaissent après refonte ou mise à jour.
- Lien : prolonge l’objectif 2030 ; prépare la nécessité d’un périmètre transverse.
- Risque à éviter : culpabiliser une équipe plutôt que montrer un mécanisme.

## Slide 3 - Poser le périmètre transverse

- Section illustrée : `Un sujet transverse`.
- Rôle dans la colonne vertébrale : clarifier le niveau de responsabilité.
- Idée principale : la contribution SNUM doit s’articuler avec des leviers MEF.
- Scène concrète visible : une table de travail avec des dossiers `projets`, `achats`, `contenus`, `recette`, reliés à un dossier central `contribution SNUM proposée`.
- Personnage ou objet-système : table de coordination.
- Point de vue incarné : décideur et référent accessibilité.
- Transformation visible : sujet MiWeb isolé -> contribution SNUM proposée -> leviers MEF à consolider.
- Texte exact : titre `Articuler la contribution SNUM avec les leviers MEF` ; labels `Projets`, `Achats`, `Contenus`, `Recette`, `Run` ; callout `Le périmètre reste à clarifier avant engagement ministériel`.
- Lexique de slide : titre-action ; labels métiers. Mots évités : `transverse` seul -> `projets, achats, contenus, recette, run`.
- Ce que l’image seule doit faire comprendre : l’accessibilité dépasse une mission web et concerne plusieurs leviers.
- Alternative textuelle provisoire : table de coordination où plusieurs leviers numériques sont reliés à une contribution SNUM proposée, avec un périmètre MEF encore à consolider.
- Lien : après le problème, on montre le bon niveau d’action ; prépare les trois axes.
- Risque à éviter : laisser croire que le SNUM porte seul tout l’engagement MEF.

## Slide 4 - Axe 1 : cadrer et prioriser

- Section illustrée : `Axe 1 - Cadrer, rendre visible, prioriser`.
- Rôle dans la colonne vertébrale : premier levier opérationnel.
- Idée principale : rendre le sujet pilotable par les parcours critiques.
- Scène concrète visible : une carte de services avec quelques parcours critiques surlignés et un poste de contrôle qui trie les priorités.
- Personnage ou objet-système : carte des parcours critiques.
- Point de vue incarné : référent accessibilité et agent métier.
- Transformation visible : parc dispersé -> parcours essentiels repérés -> priorités visibles.
- Texte exact : titre `Prioriser les parcours critiques` ; labels `Services`, `Déclarations`, `Rôles`, `Indicateurs`, `Priorités` ; callout `Rendre visible avant de généraliser`.
- Lexique de slide : titre-action ; labels structurants. Mots évités : `pilotage` -> `parcours critiques`, `priorisation` vague -> `priorités visibles`.
- Ce que l’image seule doit faire comprendre : il faut choisir les parcours où l’absence de blocage compte le plus.
- Alternative textuelle provisoire : carte de services numériques où les parcours essentiels sont identifiés, reliés à des déclarations, rôles et indicateurs.
- Lien : premier axe ; prépare l’intégration dans les projets et achats.
- Risque à éviter : suggérer que le reste du parc est abandonné.

## Slide 5 - Axe 2 : intégrer en amont

- Section illustrée : `Axe 2 - Industrialiser dans les projets et les achats`.
- Rôle dans la colonne vertébrale : deuxième levier opérationnel.
- Idée principale : l’accessibilité doit entrer dans les processus ordinaires.
- Scène concrète visible : une maquette entre dans une chaîne stable : cadrage, marché, composant, recette, livraison.
- Personnage ou objet-système : chaîne projet et achat.
- Point de vue incarné : équipe projet et acheteur.
- Transformation visible : correction tardive -> exigences en amont -> livraison mieux contrôlée.
- Texte exact : titre `Intégrer l’accessibilité en amont` ; labels `Cadrage`, `Achat`, `Maquette`, `Recette`, `Livraison` ; callout `Les outils aident, ils ne remplacent pas l’expertise`.
- Lexique de slide : titre-action ; labels de processus. Mots évités : `industrialiser` seul -> `cadrage, achat, maquette, recette, livraison`.
- Ce que l’image seule doit faire comprendre : l’accessibilité doit être traitée avant la mise en production.
- Alternative textuelle provisoire : chaîne projet montrant l’accessibilité intégrée dès le cadrage, les achats, les maquettes, la recette et la livraison.
- Lien : après la priorisation, on sécurise la fabrication ; prépare le run.
- Risque à éviter : présenter un outil de préaudit comme décidé ou suffisant.

## Slide 6 - Axe 3 : installer le run

- Section illustrée : `Axe 3 - Installer le run et l’amélioration continue`.
- Rôle dans la colonne vertébrale : troisième levier opérationnel.
- Idée principale : un service accessible doit rester accessible après les évolutions.
- Scène concrète visible : boucle de suivi autour d’un service publié : tester, détecter, corriger, actualiser, réinjecter.
- Personnage ou objet-système : boucle de maintien en condition accessible.
- Point de vue incarné : équipe de maintien en condition opérationnelle.
- Transformation visible : service publié -> évolution -> test -> correction -> retour projet.
- Texte exact : titre `Maintenir les parcours dans le temps` ; labels `Tester`, `Détecter`, `Corriger`, `Actualiser`, `Réinjecter` ; callout `Le run évite que la dette revienne`.
- Lexique de slide : titre-action ; verbes d’action. Mots évités : `amélioration continue` seule -> `tester, détecter, corriger, actualiser, réinjecter`.
- Ce que l’image seule doit faire comprendre : l’accessibilité est un fonctionnement durable, pas un jalon unique.
- Alternative textuelle provisoire : boucle de maintien montrant un service publié qui est testé, corrigé, actualisé et réinjecté dans les projets.
- Lien : conclut les trois axes ; prépare la lecture des appuis et risques.
- Risque à éviter : faire croire à une surveillance entièrement automatisée.

## Slide 7 - Cartographier appuis et risques

- Section illustrée : `Points d’appui, opportunités, fragilités et risques`.
- Rôle dans la colonne vertébrale : rendre l’arbitrage lisible.
- Idée principale : il existe des appuis solides, mais aussi des fragilités de moyens et de responsabilité à assumer.
- Scène concrète visible : une balance d’arbitrage avec, d’un côté, les appuis existants et les compétences mobilisables, de l’autre, les fragilités à assumer.
- Personnage ou objet-système : balance appuis-risques.
- Point de vue incarné : décideur.
- Transformation visible : liste dispersée -> balance lisible -> décision éclairée.
- Texte exact : titre `Lire les appuis et les risques ensemble` ; labels `RGAA`, `DSFR`, `Compétences`, `Parc hétérogène`, `Responsabilités`, `Moyens` ; callout `Les risques encadrent l’ambition`.
- Lexique de slide : titre-action ; labels précis. Mots évités : `opportunités` seules -> `RGAA, DSFR, compétences`, `fragilités` seules -> `parc hétérogène, responsabilités, moyens`.
- Ce que l’image seule doit faire comprendre : l’ambition est crédible seulement si les risques sont visibles.
- Alternative textuelle provisoire : balance d’arbitrage mettant en regard le cadre existant, les compétences mobilisables et les risques à traiter avant décision.
- Lien : après les axes, on montre les conditions de crédibilité ; prépare les conditions de réussite.
- Risque à éviter : transformer les risques en alerte anxiogène ou décorative.

## Slide 8 - Fixer les conditions de réussite

- Section illustrée : `Conditions de réussite`.
- Rôle dans la colonne vertébrale : nommer ce qui rend la trajectoire tenable.
- Idée principale : quatre conditions doivent rester visibles pour éviter une promesse vague, dont la montée en compétence.
- Scène concrète visible : quatre portes de passage avant l’horizon 2030, chacune portée par un acteur, avec une porte dédiée aux outils et compétences.
- Personnage ou objet-système : portes de réussite.
- Point de vue incarné : décideur, référent accessibilité, équipe projet, acheteur.
- Transformation visible : ambition générale -> quatre conditions -> trajectoire tenable.
- Texte exact : titre `Rendre la trajectoire tenable` ; labels `Usage prioritaire`, `Processus ordinaires`, `Gouvernance explicite`, `Outils et compétences` ; callout `Les outils aident, la montée en compétence reste centrale`.
- Lexique de slide : titre-action ; labels contrôlés. Mots évités : `gouvernance` seule -> `gouvernance explicite`, `outils` seuls -> `outils et compétences`.
- Ce que l’image seule doit faire comprendre : l’objectif 2030 dépend de conditions concrètes et de compétences installées dans la durée.
- Alternative textuelle provisoire : quatre portes représentant la priorisation par l’usage, l’intégration dans les processus, la gouvernance et l’articulation entre outils numériques et montée en compétence.
- Lien : transforme la carte des risques en conditions d’action ; prépare les décisions.
- Risque à éviter : faire croire que ces conditions sont déjà actées et financées.

## Slide 9 - Clarifier avant engagement

- Section illustrée : `Un sujet transverse` ; `Points d’appui, opportunités, fragilités et risques` ; `Conditions de réussite`.
- Rôle dans la colonne vertébrale : rendre les clarifications préalables explicites.
- Idée principale : la suite dépend de clarifications de périmètre, de responsabilités, de moyens et d’ambition avant engagement.
- Scène concrète visible : une table de clarification avec quatre dossiers ouverts et une trajectoire d’engagement encore en attente.
- Personnage ou objet-système : table de décision.
- Point de vue incarné : décideur.
- Transformation visible : trajectoire proposée -> dossiers à clarifier -> engagement seulement après périmètre confirmé.
- Texte exact : titre `Clarifier avant d’engager` ; labels `Périmètre`, `Responsabilités`, `Moyens`, `Ambition` ; callout `Le niveau SNUM/MEF reste à confirmer`.
- Lexique de slide : titre-action ; labels d’arbitrage. Mots évités : `décision` générique -> `périmètre, responsabilités, moyens, ambition`.
- Ce que l’image seule doit faire comprendre : le document prépare un engagement prudent, conditionné par les clarifications.
- Alternative textuelle provisoire : table de clarification avec quatre dossiers à consolider avant engagement du niveau SNUM ou MEF.
- Lien : après les conditions, on montre ce qui reste à clarifier ; prépare les indicateurs.
- Risque à éviter : présenter les arbitrages comme déjà tranchés.

## Slide 10 - Consolider les indicateurs de suivi

- Section illustrée : `Indicateurs de suivi`.
- Rôle dans la colonne vertébrale : terminer sur un pilotage prudent.
- Idée principale : les indicateurs doivent rester progressifs et dépendre du périmètre retenu, avec compétences et retours terrain.
- Scène concrète visible : tableau de bord en construction avec familles d’indicateurs, sans chiffres figés, incluant compétences et tests utilisateurs.
- Personnage ou objet-système : tableau de bord progressif.
- Point de vue incarné : référent accessibilité et décideur.
- Transformation visible : indicateurs dispersés -> familles consolidées -> suivi sans engagement chiffré prématuré.
- Texte exact : titre `Suivre sans figer trop tôt` ; labels `Parcours critiques`, `Déclarations/SPAN/PAN`, `Dette`, `Projets-achats`, `Compétences`, `Tests et signalements` ; callout `Mesurer dépend du périmètre confirmé`.
- Lexique de slide : titre-action ; labels de familles. Mots évités : `pilotage` seul -> `familles d’indicateurs`, `mesure` seule -> `périmètre confirmé`.
- Ce que l’image seule doit faire comprendre : la trajectoire devient pilotable si les familles d’indicateurs sont consolidées.
- Alternative textuelle provisoire : tableau de bord progressif présentant les familles d’indicateurs de suivi, y compris compétences, accompagnement, tests utilisateurs et signalements, sans chiffre définitif.
- Lien : clôture le récit sur la prudence et la capacité de suivi.
- Risque à éviter : inventer des seuils ou objectifs chiffrés.
